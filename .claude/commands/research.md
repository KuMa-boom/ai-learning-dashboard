以下の入力を受け取り、調査してわかりやすくまとめてください。

**入力：** $ARGUMENTS

---

## 処理ルール

### GitHub Issue URLが渡された場合

URLが `https://github.com/` から始まり、`/issues/` を含む場合：

1. GitHub MCPツール（`mcp__github__*` 系）が使えるなら優先して使う
2. MCPが使えない場合は `gh issue view <番号> --repo <owner/repo>` で取得する
3. 以下を整理してレポートする：
   - **Issue概要**：タイトル・番号・状態（open/closed）・作成者・作成日
   - **内容まとめ**：何が問題か・何を求めているか（箇条書き）
   - **コメント・議論**：重要な議論や決定事項
   - **ラベル・マイルストーン**：あれば記載
   - **関連PR**：紐づいているPRがあれば番号と状態

### それ以外の文字列が渡された場合

1. まず入力がプロジェクト内のコードや設定に関するトピックかどうか判断する
2. **プロジェクト関連の場合**：
   - Glob・Grep・Readを使ってコードベースを調査する
   - 関連ファイル・関数・設定を特定してまとめる
3. **一般的なトピックの場合**：
   - WebFetchで信頼できる情報源を調べる
   - 調査結果を日本語でまとめる

---

## 出力形式

- 日本語で出力する
- Markdownの見出し・箇条書きを使って読みやすく整形する
- 長い場合はセクションに分けて構造化する
- 不明な点や確認できなかった点は正直に記載する
